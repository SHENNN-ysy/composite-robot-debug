# 机器人控制系统 TCP 模拟器

仅用于本地联调的 TCP 服务端，模拟机器人控制系统向 Java 后端单向发送机械臂状态。

## 协议约定

- 后端作为 TCP 客户端连接 `127.0.0.1:9000`
- 服务端只推送不接收；每帧数据采用 UTF-8 换行 JSON
- 字段必须包含：x/y/z/rx/ry/rz/j1~j6/io(DI/DO/CI/CO 各 8 位布尔) + running/speed/mode
- 字段缺失或类型错误的整帧会被后端丢弃

详细协议见 `backend/TCP_PROTOCOL.md`。

## 使用方式

```bash
# 依赖：Python 3.8+（仅使用标准库）
python robot_controller_simulator.py
```

启动后控制台交互菜单：

```
===== 机器人控制系统 TCP 模拟器 =====
1. 启动 TCP 服务端
2. 启动 + 加载预设场景
3. 退出
```

服务启动后，按 `Ctrl+C` 退出；运行中可在标准输入键入快捷键：

| 按键 | 行为 |
|------|------|
| `s` | 打印当前帧状态 |
| `p` | 暂停 / 恢复推送 |
| `m` | 切换运动模式（手动 / 自动 / 待机） |
| `q` | 断开客户端连接（模拟断线） |
| `h` | 打印帮助 |

## 预设场景

`2. 启动 + 加载预设场景` 时可选：

| 场景 | 行为 |
|------|------|
| `static` | 持续输出固定姿态（默认） |
| `circles` | TCP 位置沿圆周缓慢运动 |
| `joint_swing` | J2/J3 同时正弦摆动 |
| `io_blink` | DO[0] 每 0.5 秒翻转一次 |
| `full_run` | 模拟自动运行：位置循环 + 速度变化 + DO[0] 节拍 |
